Premier Protein Retail Analysis

A SQL‑based exploration of retailer pricing, pack sizes, flavors, and promotions for Premier Protein listings. Designed as a focused fundamentals project with clean, minimal presentation.

Project Overview

This project analyzes Premier Protein retail listings across multiple retailers using SQL Server Developer Edition. The goal is to practice core SQL fundamentals—aggregation, joins, filtering, and granularity.

The dataset includes catalog, product, flavor, pack size, retailer, and promotion tables, allowing targeted questions about pricing behavior, listing distribution, and promotional activity.

Dataset & Tools

Tables used in this project:

  • fact_catalog — core listing and price data
  • dim_products — product and brand details
  • dim_flavors — flavor names and IDs
  • dim_pack_sizes — pack size definitions
  • dim_retailers — retailer dimension
  • fact_promotions — promotion events and promo prices

Tools used:

SQL Server Developer Edition T‑SQL GitHub Pages
SQL Fundamentals Showcase

Five out of six core SQL questions completed, each focused on a specific concept: aggregation, joins, filtering, and granularity.

1. Listings per retailer
Counts how many listings each retailer has in the catalog using COUNT(*) and GROUP BY retailer_id.
2. Average price by pack size
Calculates the average price for each pack_size_id across all retailers using AVG(price) and GROUP BY pack_size_id.
3. Target listings with product & flavor names
Returns all listings sold at Target (retailer_id = 4), including product and flavor names using JOIN across catalog, products, flavors, and retailers with a WHERE filter.
4. Promotions summary per retailer
Calculates the total number of promotions and sum of promo_price for each retailer using COUNT(*), SUM(promo_price), and GROUP BY retailer_id.
5. Additional fundamentals query
Return all listings from Kroger where the price is greater than $10.00.
Key Insights
  • Retailers show distinct listing volumes, highlighting differences in Premier Protein shelf presence.
  • Average price varies by pack size, suggesting a relationship between unit count and perceived value.
  • Target listings can be cleanly isolated and enriched with product and flavor details using joins.
  • Promotional activity clusters by retailer, visible through promotion counts and total promo_price.
